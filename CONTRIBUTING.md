# Contributing to cint

Thank you for your interest in contributing to cint! Your help is greatly appreciated. Please follow these guidelines to help us maintain a welcoming and productive environment.

## How to Contribute

1. **Fork the repository** and create your branch from `main`.
2. **Write clear, concise commit messages**.
3. **Test your code** before submitting a pull request.
4. **Open a pull request** with a clear description of your changes and why they are needed.
5. **Participate in code review** and address any feedback.

## Code Style

- Follow idiomatic Go style. Use `gofmt` to format your code.
- Write clear, maintainable, and well-documented code.
- Add comments where necessary, especially for complex logic.

## Issues

- Search existing issues before opening a new one.
- When reporting a bug, include steps to reproduce, expected behavior, and actual behavior.
- For feature requests, describe the motivation and use case.

## Pull Requests

- Keep pull requests focused and small. Large or unrelated changes may be rejected.
- Reference related issues in your pull request description.
- Ensure all tests pass before requesting a review.

## Community Standards

- Be respectful and inclusive. See our [Code of Conduct](CODE_OF_CONDUCT.md).
- Constructive feedback and collaboration are encouraged.
